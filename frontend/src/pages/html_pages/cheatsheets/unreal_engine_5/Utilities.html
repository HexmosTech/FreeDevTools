Utilities

Delay

There is no equivalent to the delay node in C++, so timers are used instead. If you want to execute code after a specific delay, put that code into a function and then start a timer with the desired delay. See: Set Timer by Function Name

Print String

Commonly used, but has a few minor differences to the blueprint node:

if (GEngine)
{
    GEngine->AddOnScreenDebugMessage(
        -1,                        // Key
        2.0f,                      // Duration
        FColor::Cyan,              // Color
        "Hello"                    // Message
    );
}

The exact blueprint node:

#include "Kismet/KismetSystemLibrary.h"

UKismetSystemLibrary::PrintString(
    this,                                    // World context object
    "Hello",                                 // Message
    true,                                    // Print to screen?
    true,                                    // Print to console?
    FLinearColor(0.0f, 0.66f, 1.0f, 1.0f),   // Color
    2.0f                                     // Duration
);

Is Valid

Not identical to the blueprint node but used in most cases:

AActor* ExampleActor;

// Check validity
if (ExampleActor)
{
    // Is Valid...
}
else
{
    // Is Not Valid...
}

Identical to the blueprint node:

#include "Kismet/KismetSystemLibrary.h"

AActor* ExampleActor;

// Check validity
if (UKismetSystemLibrary::IsValid(ExampleActor))
{
    // Is Valid...
}
else
{
    // Is Not Valid...
}

Quit Game

#include "Kismet/GameplayStatics.h"
#include "Kismet/KismetSystemLibrary.h"

// Get player controller
APlayerController* PlayerController = UGameplayStatics::GetPlayerController(this, 0);

// Quit game
UKismetSystemLibrary::QuitGame(
    this,                        // World context object
    PlayerController,            // Quitting player controller
    EQuitPreference::Quit,       // Quit preference
    false                        // Ignore platform restrictions?
);
