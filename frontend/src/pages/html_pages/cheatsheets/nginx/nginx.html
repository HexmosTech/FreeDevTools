Essential Nginx Configurations

Here are some fundamental Nginx configuration examples to get you started:

Bare Bones Website Configuration

A minimal server block to serve static content from a specified directory.

server {
    listen       80;
    server_name  localhost;

    location / {
        root   /usr/share/nginx/html;
        index  index.html index.htm;
    }

    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }
}

Redirecting Non-existent Pages to Homepage

Configure Nginx to return a 301 redirect to the root URL for any 404 Not Found errors.

    # Define the error page for 404
    error_page 404 = @notfound;

    # Define the named location for the redirect
    location @notfound {
        return 301 /;
    }

Redirecting Old URLs to New Paths

Use the rewrite directive to map old URL paths to new file locations on disk.

    # Redirect old URLs to new paths
    location /content/images/2019/10/logo.png {
        rewrite ^/content/images/2019/10/logo.png /assets/img/logo.png permanent;
    }

Understanding Nginx Location Directives

The location directive is fundamental in Nginx for matching request URIs and applying specific configurations. Proper understanding of its priority and syntax is key to effective Nginx setup. For instance, exact matches (=) have the highest priority, followed by prefix matches (^~), then regular expressions (~ and ~*), and finally longest prefix matches.

HTTP Request Handling

Nginx is highly capable of handling different HTTP methods like GET and POST. It can proxy requests to backend servers, serve static files, and manage complex routing logic. Ensuring correct handling of request methods prevents data loss and ensures smooth application functionality.
