NAME
       tracereplay - Replay a trace in trace time from one uri to another


SYNOPSIS
       tracereplay  [-b  |  --broadcast]  [-s  --snaplength  [  snaplength] ] [-f | --filter [ filter string ] ]
       inputuri outputuri


DESCRPTION
       tracereplay replays inputuri to outputuri in trace time. Checksums are recomputed on the fly.

       -b
       --broadcast

              Change the 802.3 (ethernet) destination address to the broadcast  address.   This  will  send  the
              transmitted packets to any hosts on the local network.

       -s[snaplength]
       --snaplength[snaplength]
              Truncate  the  packets  to the length specified. This only makes a difference if the inputuri is a
              capture device. The value should be > 0. Inappropriate values will be ignored, but a warning  will
              be printed.

       -f[filterstring]
       --filter[filter]
              Apply a filter to the inputuri.
