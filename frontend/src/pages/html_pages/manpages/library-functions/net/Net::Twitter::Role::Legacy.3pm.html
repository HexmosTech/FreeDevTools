NAME
       Net::Twitter::Role::Legacy - A Net::Twitter legacy compatibility layer as a Moose role


VERSION
       version 4.01043


SYNOPSIS
           use Net::Twitter;

           my $nt = Net::Twitter->new(
               username => $username,
               password => $password,
               traits   => [qw/Legacy/],
           );

           my $followers = $nt->followers;
           if ( !followers ) {
               warn $nt->http_message;
           }


DESCRIPTION
       This module provides a Net::Twitter compatibility layer for Net::Twitter.  It pulls in the additional
       traits: "API::REST", "API::Search", "API::Identica", and "WrapError".


METHODS
       new This  method  takes  the  same parameters as "new" in Net::Twitter.  In addition, it also support the
           options:

           arrayref_on_error
               When set to 1, on error, rather than returning undef, the API methods will return an empty  ARRAY
               ref.  Defaults to 0.

           twittervision
               When set to 1, enables the "upade_twittervision" call.  Defaults to 0.

       clone
           Creates  a  shallow  copy  of  the  "Net::Twitter"  object.   This  was useful, in legacy versions of
           "Net::Twitter" for handling concurrent  requests  (for  instance  with  LWP::UserAgent::POE).   Since
           errors  are  wrapped in the "Net::Twitter" concurrent requests each needed their own object.  "clone"
           served that purpose.

           The recommended approach for concurrent requests is to use "Net::Twitter"'s ability throw exceptions,
           now.
