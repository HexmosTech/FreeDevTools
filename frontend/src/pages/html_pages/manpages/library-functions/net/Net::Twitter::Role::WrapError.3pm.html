NAME
       Net::Twitter::Role::WrapError - Wraps Net::Twitter exceptions


VERSION
       version 4.01043


SYNOPSIS
           use Net::Twitter;

           my $nt = Net::Twitter->new(username => $username, password => $password);

           my $followers = $nt->followers;
           if ( !followers ) {
               warn $nt->http_message;
           }


DESCRIPTION
       This module provides an alternate error handling strategy for "Net::Twitter".  Rather than throwing
       exceptions, API methods return "undef" and error information is available through method calls on the
       "Net::Twitter" object.

       This is the error handling strategy used when "trait" "Legacy" is used.  It was the error handling
       strategy employed by "Net::Twitter" prior to version 3.00.


METHODS
       new This method takes the same parameters as "new" in Net::Twitter.

       get_error
           Returns  undef  if  there  was  no  error  or  a  HASH  ref  containing  the  Twitter error response.
           Occasionally, a Twitter API call results in an error with no error  content  returned  from  Twitter.
           When that occurs, get_error returns a simulated error HASH ref.

           NOTE:  Versions of "Net::Twitter" prior to 3.0 sometimes returned a string rather than a HASH ref, on
           error.  This was a bug.  Always expect undef on success and a HASH ref on error.

       http_code
           Returns the HTTP response code the most recent API method call if it ended in error.

       http_message
           Returns the HTTP message for the most recent API method call if it ended in error.
