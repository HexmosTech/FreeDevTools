NAME
       Test2::Bundle - Documentation for bundles.


DESCRIPTION
       Bundles are collections of Tools and Plugins. Bundles should not provide any tools or behaviors of their
       own, they should simply combine the tools and behaviors of other packages.


FAQ
       Should my bundle subclass Test2::Bundle?
           No.  Currently  this  class is empty. Eventually we may want to add behavior, in which case we do not
           want anyone to already be subclassing it.


HOW DO I WRITE A BUNDLE?
       Writing a bundle can be very simple:

           package Test2::Bundle::MyBundle;
           use strict;
           use warnings;

           use Test2::Plugin::ExitSummary; # Load a plugin

           use Test2::Tools::Basic qw/ok plan done_testing/;

           # Re-export the tools
           our @EXPORTS = qw/ok plan done_testing/;
           use base 'Exporter';

           1;

       If you want to do anything more complex you should look into Import::Into and Symbol::Move.
