NAME
       Test2::Harness::Collector - Module that collects test output and provides it as an event stream.


DESCRIPTION
       This module is responsible for reading and parsing the output produced by multiple jobs running under
       yath.

       This module is not intended for external use, it is an implementation detail and can change at any time.
       Currently instances of this module are not passed to any plugins or callbacks.

       If you need a collector for a third-party command you should look at App::Yath::Command::collector. When
       a command needs a collector (such as App::Yath::Command::test does) it normally spawns a collector
       process by execuing "yath collector". The start_collector() subroutine in App::Yath::Command::test is a
       good place to look for more details.
