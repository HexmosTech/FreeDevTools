NAME
       SVN::Hooks::CheckProperty - Check properties in added files.


VERSION
       version 1.36


SYNOPSIS
       This SVN::Hooks plugin checks if some files added to the repository have some properties set.

       It's active in the "pre-commit" hook.

       It's configured by the following directive.

   CHECK_PROPERTY(WHERE, PROPERTY[, VALUE])
       This directive enables the checking, causing the commit to abort if it doesn't comply.

       The WHERE argument must be a qr/Regexp/ matching all files that must comply to this rule.

       The PROPERTY argument is the name of the property that must be set for the files matching WHERE.

       The optional VALUE argument specifies the value for PROPERTY depending on its type:

       UNDEF or not present
           The PROPERTY must be set.

       NUMBER
           If non-zero, the PROPERTY must be set. If zero, the PROPERTY must NOT be set.

       STRING
           The PROPERTY must be set with a value equal to the string.

       qr/Regexp/
           The PROPERTY must be set with a value that matches the Regexp.

       Example:

               CHECK_PROPERTY(qr/\.(?:do[ct]|od[bcfgimpst]|ot[ghpst]|pp[st]|xl[bst])$/i
                      => 'svn:needs-lock');
