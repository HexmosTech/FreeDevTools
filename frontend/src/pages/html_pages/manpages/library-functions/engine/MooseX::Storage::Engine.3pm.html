NAME
       MooseX::Storage::Engine - The meta-engine to handle collapsing and expanding objects


VERSION
       version 0.53


DESCRIPTION
       There really aren't any major user serviceable parts here. However the typical use case is adding new
       non-Moose classes to the type registry for serialization. Here is an example of this for DateTime
       objects. This assumes a "DateTime" type has been registered.

           MooseX::Storage::Engine->add_custom_type_handler(
               'DateTime' => (
                   expand   => sub { DateTime::Format::ISO8601->new->parser_datetime(shift) },
                   collapse => sub { (shift)->iso8601 },
               )
           );
