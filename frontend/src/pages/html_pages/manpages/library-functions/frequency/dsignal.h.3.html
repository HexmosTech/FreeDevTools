Detailed Description
       Contains the API for serializing signals to a pipe for usage with select() or poll().

       You should register all signals you wish to handle with select() in your main loop with
       daemon_signal_init() or daemon_signal_install(). After that you should sleep on the file descriptor
       returned by daemon_signal_fd() and get the next signal received with daemon_signal_next(). You should
       call daemon_signal_done() before exiting.

   int daemon_signal_fd (void)
       Return the file descriptor the daemon should select() on for reading. Whenever the descriptor is ready
       you should call daemon_signal_next() to get the next signal queued.

       Return the next signal received. This function will not block. Instead it returns 0 if no signal is
