NAME
       Plack::Middleware::Proxy::RewriteLocation - Rewrites redirect headers


SYNOPSIS
         use Plack::Builder;
         use Plack::App::Proxy;

         builder {
             enable "Proxy::RewriteLocation";
             Plack::App::Proxy->new(remote => "http://10.0.1.2:8080/")->to_app;
         };

         ### or, if mounting (i.e. URLMap) the proxied site at /foo

         builder {
             enable "Proxy::RewriteLocation", url_map => [ '/foo' => http://10.0.1.2:8080' ];
             mount '/foo' => Plack::App::Proxy->new(remote => "http://10.0.1.2:8080/")->to_app;
         };


DESCRIPTION
       Plack::Middleware::Proxy::RewriteLocation rewrites the "Location" header in the response when the remote
       host redirects using its own headers, like mod_proxy's "ProxyPassReverse" option.


OPTIONS
       url_map (arrayref)
           If given, will account for mounted (URLMapped) Proxy apps when rewriting "Location" headers.  Will be
           applied in order, stopping at the first successful match with the remote "Location".
