NAME
       Plack::Middleware::RearrangeHeaders - Reorder HTTP headers for buggy clients


SYNOPSIS
         use Plack::Builder;

         my $app = sub {
             return [ 200, [
                 'Last-Modified' => 'Wed, 23 Sep 2009 13:36:33 GMT',
                 'Content-Type' => 'text/plain',
                 'ETag' => 'foo bar',
             ], [ 'Hello Foo' ] ];
         };

         builder {
             enable "Plack::Middleware::RearrangeHeaders";
             $app;
         };


DESCRIPTION
       Plack::Middleware::RearrangeHeaders sorts HTTP headers based on "Good Practice" i.e.:

         # "Good Practice" order of HTTP message headers:
         #    - Response-Headers
         #    - Entity-Headers

       to work around buggy clients like very old MSIE or broken HTTP proxy servers. Most clients today don't
       (and shouldn't) care about HTTP header order but if you're too pedantic or have some environments where
       you need to deal with buggy clients like above, this might be useful.
