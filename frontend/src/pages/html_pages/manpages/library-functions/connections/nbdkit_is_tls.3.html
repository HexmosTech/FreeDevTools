NAME
       nbdkit_is_tls - get TLS status of the connection for nbdkit


SYNOPSIS
        #include <nbdkit-plugin.h>

        int nbdkit_is_tls (void);


DESCRIPTION
       "nbdkit_is_tls" returns true if and only if TLS negotiation has been completed on the current connection.
       An nbdkit(1) plugin may use "nbdkit_is_tls" to limit connections according to whether the client
       connection is protected by TLS.  This function uses thread-local magic so no parameter is required

       nbdkit-tls-fallback-filter(1) and the nbdkit --tls=require option are other alternatives.


RETURN VALUE
       The function returns 1 (true) if TLS negotiation has been completed, or 0 (false) if not.

       If there is an error it calls nbdkit_error(3) and returns -1.


LANGUAGE BINDINGS
       In nbdkit-ocaml-plugin(3):

        NBDKit.is_tls : unit -> bool

       In nbdkit-python-plugin(3):

        import nbdkit
        tls = nbdkit.is_tls()

       In nbdkit-rust-plugin(3):

        use nbdkit::*;
        pub fn is_tls() -> Result<bool>

       In nbdkit-sh-plugin(3) the TLS flag is available as parameter $4 of the "open" method.


HISTORY
       "nbdkit_is_tls" was added in nbdkit 1.22.
