NAME
       Test::Effects::TIME - Lexically set Test::Effects timing option


VERSION
       This document describes Test::Effects::TIME version 0.000001


SYNOPSIS
           {
               use Test::Effects::TIME;
               # Test::Effects times its tests

               {
                   no Test::Effects::TIME;
                   # Test::Effects doesn't time its tests here
               }

               # Test::Effects reverts to timing its tests again here

           }
           # Test::Effects reverts to original (non-timing) behaviour here


DESCRIPTION
       Using this module changes the default behaviour of Test::Effects's "effects_ok()" test.

       Normally this test does not time the code it tests (unless explicitly told otherwise via the use of "TIME
       => 1" or "TIME()"). If this module is used in a code block, "effects_ok()" defaults to timing its tests
       for the remainder of that block's lexical scope.

       Note, however, that an explicit "TIME => 0" option in any call to "<effects_ok()" overrides this lexical
       default.


INTERFACE
   "use Test::Effects::TIME;"
       "effects_ok" defaults to timing the code it tests within the rest of the current lexical scope.

   "no Test::Effects::TIME;"
       "effects_ok" defaults to not timing the code it tests within the rest of the current lexical scope.

CONFIGURATION AND ENVIRONMENT

DEPENDENCIES
