NAME
       SNMP::Info::Layer3::OneAccess - SNMP Interface to OneAccess routers.


AUTHORS
       Rob Woodward


SYNOPSIS
        # Let SNMP::Info determine the correct subclass for you.
        my $oneos = new SNMP::Info(
                                 AutoSpecify => 1,
                                 Debug       => 1,
                                 DestHost    => 'myrouter',
                                 Community   => 'public',
                                 Version     => 2
                               )
           or die "Can't connect to DestHost.\n";

        my $class      = $oneos->class();
        print "SNMP::Info determined this device to fall under subclass : $class\n";


DESCRIPTION
       Subclass for OneAccess routers.

   Inherited Classes
       SNMP::Info::Layer3

   Required MIBs
       ONEACCESS-GLOBAL-REG
       ONEACCESS-SYS-MIB

   Inherited Classes' MIBs
           See SNMP::Info::Layer3 for its own MIB requirements.


GLOBALS
       These are methods that return scalar value from SNMP.

       $oneos->oa_model()
           Returns the hardware model from "oacExpIMSysHwcProductName.0".

       $oneos->os_ver()
           Returns the software version extracted from "sysDescr".

   Overrides
       $oneos->model()
           Returns oa_model() with a fallback to e_model() from SNMP::Info::Entity.

       $oneos->os()
           Returns 'oneos'.

       $oneos->vendor()
           Returns 'oneaccess'.

   Globals imported from SNMP::Info::Layer3
       See documentation in SNMP::Info::Layer3 for details.


TABLE ENTRIES
       These are methods that return tables of information in the form of a reference to a hash.

   Table Methods imported from SNMP::Info::Layer3
       See documentation in SNMP::Info::Layer3 for details.
