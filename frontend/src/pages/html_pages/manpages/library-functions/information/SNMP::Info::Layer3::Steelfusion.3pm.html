NAME
       SNMP::Info::Layer3::Steelfusion - SNMP Interface to Riverbed Steelhead WAN optimization appliances.


AUTHOR
       Eric Miller


SYNOPSIS
        # Let SNMP::Info determine the correct subclass for you.
        my $riverbed = new SNMP::Info(
                                 AutoSpecify => 1,
                                 Debug       => 1,
                                 DestHost    => 'myswitch',
                                 Community   => 'public',
                                 Version     => 2
                               )
           or die "Can't connect to DestHost.\n";

        my $class = $riverbed->class();
        print "SNMP::Info determined this device to fall under subclass : $class\n";


DESCRIPTION
       Abstraction subclass for Riverbed Steelhead WAN optimization appliances.

   Inherited Classes
       SNMP::Info::Layer3

   Required MIBs
       STEELHEAD-MIB

       Inherited Classes' MIBs
           See "Required MIBs" in SNMP::Info::Layer3 for its own MIB requirements.


GLOBALS
       These are methods that return scalar value from SNMP

       $riverbed->vendor()
           Returns 'riverbed'

       $riverbed->model()
           Returns the chassis model.

           ("STEELHEAD-MIB::model")

       $riverbed->os()
           Returns 'steelhead'

       $riverbed->os_ver()
           Returns the software version extracted from ("systemVersion").

       $riverbed->serial()
           Returns the chassis serial number.

           ("serialNumber")

   Overrides
       $riverbed->layers()
           Returns 01001100.  Steelhead does not support bridge MIB, so override reported layers.

   Globals imported from SNMP::Info::Layer3
       See documentation in "GLOBALS" in SNMP::Info::Layer3 for details.
