NAME
       SDL_HINT_FILE_DIALOG_DRIVER - A variable that specifies a dialog backend to use.


HEADER FILE
       Defined in SDL3/SDL_hints.h


SYNOPSIS
       #include "SDL3/SDL.h"

       #define SDL_HINT_FILE_DIALOG_DRIVER SDL_FILE_DIALOG_DRIVER"


DESCRIPTION
       By  default, SDL will try all available dialog backends in a reasonable order until it finds one that can
       work, but this hint allows the app or user to force a specific target.

       If the specified target does not exist or is not available, the dialog-related function calls will fail.

       This hint currently only applies to platforms using the generic "Unix" dialog implementation, but may  be
       extended  to  more  platforms  in the future.  Note that some Unix and Unix-like platforms have their own
       implementation, such as macOS and Haiku.

       The variable can be set to the following values:

       • NULL: Select automatically (default, all platforms)

       • "portal": Use XDG Portals through DBus (Unix only)

       • "zenity": Use the Zenity program (Unix only)

       More options may be added in the future.

       This hint can be set anytime.


AVAILABILITY
       This hint is available since SDL 3.2.0.
