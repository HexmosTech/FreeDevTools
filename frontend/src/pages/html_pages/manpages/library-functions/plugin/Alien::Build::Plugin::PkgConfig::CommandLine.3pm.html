NAME
       Alien::Build::Plugin::PkgConfig::CommandLine - Probe system and determine library or tool properties
       using the pkg-config command line interface


VERSION
       version 2.84


SYNOPSIS
        use alienfile;
        plugin 'PkgConfig::CommandLine' => (
          pkg_name => 'libfoo',
        );


DESCRIPTION
       Note: in most case you will want to use Alien::Build::Plugin::PkgConfig::Negotiate instead.  It picks the
       appropriate fetch plugin based on your platform and environment.  In some cases you may need to use this
       plugin directly instead.

       This plugin provides Probe and Gather steps for pkg-config based packages.  It uses the best command line
       tools to accomplish this task.


PROPERTIES
   pkg_name
       The package name.  If this is a list reference then .pc files with all those package names must be
       present.  The first name will be the primary and used by default once installed.  For the subsequent
       ".pc" files you can use the Alien::Base alt method to retrieve the alternate configurations once the
       Alien is installed.

   atleast_version
       The minimum required version that is acceptable version as provided by the system.

   exact_version
       The exact required version that is acceptable version as provided by the system.

   max_version
       The max required version that is acceptable version as provided by the system.

   minimum_version
       Alias for "atleast_version" for backward compatibility.


METHODS
   available
        my $bool = Alien::Build::Plugin::PkgConfig::CommandLine->available;

       Returns true if the necessary prereqs for this plugin are already installed.
