NAME
       Other_Data_Types - Other Built-in Data Types


DESCRIPTION
       The following table describes other data types supported by OpenCL:

       ┌──────────────────┬──────────────────────────────────────┐
       │ Type             │ Description                          │
       ├──────────────────┼──────────────────────────────────────┤
       │ image2d_t        │ A 2D image. See individual           │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ image3d_t        │ A 3D image. See individual           │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ image2d_array_t  │ A 2D image array. See individual     │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ image1d_t        │ A 1D image. See individual           │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ image1d_buffer_t │ A 1D image created from a buffer     │
       │                  │ object. See individual               │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ image1d_array_t  │ A 1D image array. See individual     │
       │                  │ imageFunctions(3clc)  for more       │
       │                  │ information.                         │
       ├──────────────────┼──────────────────────────────────────┤
       │ sampler_t(3clc)  │ A sampler type.                      │
       ├──────────────────┼──────────────────────────────────────┤
       │ event_t          │ An event. This can be used to        │
       │                  │ identify async copies from global to │
       │                  │ local memory and vice-versa. See     │
       │                  │ individual  asyncCopyFunctions(3clc) │
       │                  │ for more information.                │
       └──────────────────┴──────────────────────────────────────┘

       NOTE: The image2d_t, image3d_t, image2d_array_t, image1d_t, image1d_buffer_t, image1d_array_t, and
       sampler_t types are only defined if the device supports images i.e.  CL_DEVICE_IMAGE_SUPPORT is CL_TRUE.
       See the table for param_name for clGetDeviceInfo(3clc) for details.

       The C99 derived types (arrays, structs, union, function, and pointers), constructed from the built-in
       data types described in scalarDataTypes(3clc), vectorDataTypes(3clc), and otherDataTypes(3clc) are also
       supported with certain restrictions(3clc).
