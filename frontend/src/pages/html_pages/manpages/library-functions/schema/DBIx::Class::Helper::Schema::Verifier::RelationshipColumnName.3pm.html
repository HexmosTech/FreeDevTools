NAME
       DBIx::Class::Helper::Schema::Verifier::RelationshipColumnName - Verify that relationships and column
       names are distinct


SYNOPSIS
        package MyApp::Schema;

        __PACKAGE__->load_components('Helper::Schema::Verifier::RelationshipColumnName');


DESCRIPTION
       "DBIx::Class::Helper::Schema::Verifier::RelationshipColumnName" verifies that none of your columns have
       the same name as a relationship.  If you create a relationship that has the same name as a column, to
       access the column you will be forced to use "get_column", additionally it is just confusing having them
       be the same name.  What I tend to do is define the columns to be something like "user_id" and have the
       relationship then be simply "user".
