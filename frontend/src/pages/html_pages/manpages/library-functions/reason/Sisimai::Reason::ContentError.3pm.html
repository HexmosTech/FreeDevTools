NAME
       Sisimai::Reason::ContentError - Bounce reason is "contenterror" or not.


SYNOPSIS
           use Sisimai::Reason::ContentError;
           print Sisimai::Reason::ContentError->match('550 Message Filterd'); # 1


DESCRIPTION
       "Sisimai::Reason::ContentError" checks the bounce reason is "contenterror" or not. This class is called
       only "Sisimai::Reason" class.

       This is the error that the destination mail server has rejected email due to the header format of the
       email like the following. Sisimai will set "contenterror" to the reason of the email bounce if the value
       of "Status:" field in a bounce email is "5.6.*".

       - 8 bit data in message header
       - Too many "Received:" headers
       - Invalid MIME headers

           ... while talking to g5.example.net.:
           >>> DATA
           <<< 550 5.6.9 improper use of 8-bit data in message header
           554 5.0.0 Service unavailable


CLASS METHODS
   "text()"
       text() method returns the fixed string "contenterror".

           print Sisimai::Reason::ContentError->text;  # contenterror

   "match(string)"
       match() method returns 1 if the argument matched with patterns defined in this class.

           print Sisimai::Reason::ContentError->match('550 Message Filterd'); # 1

   "true(Sisimai::Fact)"
       true()  method  returns  1  if  the bounce reason is "contenterror". The argument must be "Sisimai::Fact"
       object and this method is called only from "Sisimai::Reason" class.
