NAME
       BackupLog - Traces Backup Server operations


DESCRIPTION
       The BackupLog file records a trace of Backup Server (buserver process) operations on the local machine
       and describes any error conditions it encounters.

       If the BackupLog file does not already exist in the /var/log/openafs directory when the Backup Server
       starts, the server process creates it and writes initial start-up messages to it.

       The file is in ASCII format. Administrators listed in the /etc/openafs/server/UserList file can use the
       bos getlog command to display its contents. Alternatively, log on to the machine and use a text editor or
       a file display command such as the UNIX cat command. By default, the mode bits on the BackupLog file
       grant the required "r" (read) permission to all users.

       The Backup Server records operations only as it completes them, and so cannot recover from failures by
       reviewing the file. The log contents are useful for administrative evaluation of process failures and
       other problems.
